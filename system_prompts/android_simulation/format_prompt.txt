Given the following content of a GUI and the reference GUI in the format of a list of UI Elements, translate the GUI content into new state, based on previous GUI. 

Each element is a struct which has the following attributes:
"text": The visible text label shown to the user (e.g. "Gmail").
"content_description": An accessibility label used by screen readers; describes the UI element when no visible text is present.
"class_name": The fully qualified Android class name that represents the widget type (e.g. "android.widget.TextView").
"bbox": A general-purpose bounding box (seldom unused).
"bbox_pixels": The elementâ€™s screen position and size as a pixel-based bounding box: (x_min, x_max, y_min, y_max).
"hint_text": Placeholder text for input fields, usually shown when the field is empty.
"is_checked": Indicates whether the element is currently in a checked state (e.g. checkbox is checked).
"is_checkable": Indicates whether the element supports being checked or unchecked.
"is_clickable": Indicates whether the element responds to user taps.
"is_editable": Indicates whether the element accepts user text input.
"is_enabled": Indicates whether the element is currently enabled and interactable.
"is_focused": Indicates whether the element currently has focus (e.g. selected for input).
"is_focusable": Indicates whether the element is capable of receiving focus.
"is_long_clickable": Indicates whether the element supports long-press gestures.
"is_scrollable": Indicates whether the element can scroll, such as a list or scroll view.
"is_selected": Indicates whether the element is selected, useful in tabs or list views.
"is_visible": Indicates whether the element is visible on the screen (not hidden or gone).
"package_name": The Android package name of the application that owns the element (e.g. "com.google.android.apps.nexuslauncher").
"resource_name": The string name of the resource associated with this element (e.g. "R.id.search_box").
"tooltip": A brief text shown as a tooltip when the user hovers or long-presses the element.
"resource_id": The internal Android resource ID used to reference the element programmatically.
"metadata": A generic field for any additional metadata associated with the element, such as analysis annotations.
